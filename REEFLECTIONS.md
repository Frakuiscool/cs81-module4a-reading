Question 1: What was the most helpful aspect of this code's structure?
The most helpful part was how the code was broken up into small, single-purpose functions, with each one handling just one task like adding an entry, totaling minutes, finding the most read book, or printing a summary. This made the code much easier to follow, since I could look at one function at a time instead of trying to understand everything at once. The comments and function summaries above each function also helped a lot, since they explained the purpose, inputs, and output before I even got into the logic itself.
Question 2: What part was confusing or took longer to understand?
The part that took me longest to understand was the difference between for...of and for...in, and why each one was used in different places. for...of loops over the values in an array (like each reading entry), while for...in loops over the keys of an object (like each book title in bookCounts). It also took me a bit to fully understand how bookCounts works as a tally system, since it's using an object almost like a dictionary to count occurrences, which was a new pattern for me. Once I traced through it step by step, though, it started to make more sense.
